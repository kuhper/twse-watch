# 台股注意／處置 三階段預判（twse_watch）

輸入一檔**上市／上櫃**股票代號，向證交所（TWSE）、櫃買中心（TPEx）官方公開資料
即時抓取，依「注意交易資訊暨處置作業要點」判定，輸出三階段結果：

1. **第一階 · 既成公告**：抓官方注意／處置公告。若已公告處置且有明確期間 → 到此結束。
2. **第二階 · 規則判定**：把官方注意／處置標準公式化，判斷今日踩到哪幾款、距處置還差幾次。
3. **第三階 · 反推數字**：反推「明日」觸發各款所需的臨界收盤價、需維持的週轉率、需放大的量能，
   並對照次日漲跌幅限制（±10%）判斷「一日內可達」或「需連續數日」。

> ⚠️ 僅供研究參考，實際以證交所／櫃買中心公告為準，非投資建議。

---

## 快速開始

```bash
# 1. 安裝相依套件（建議 Python 3.10+）
pip install -r requirements.txt

# 2. 啟動服務（請在專案根目錄執行）
python start.py
# 開啟瀏覽器 → http://127.0.0.1:8000
# 輸入代號，例如 2330、2302、2603
```

純 API 用法：

```bash
curl "http://127.0.0.1:8000/api/query?code=2330&months=6"
```

跑單元測試（合成資料，不需連網）：

```bash
python tests/test_core.py
```

## 已實作的注意標準（要點第四條第一項各款）

引擎依官方「異常標準之詳細數據及除外情形」（113/12/31 版）公式化。各款狀態會在
介面標示為「達標／未達／除外／資料不足」：

| 款 | 內容 | 狀態 |
| --- | --- | --- |
| 第一款 | 近6日累積漲跌幅 > ±30%（或 >±25% 且起迄價差≥50元） | ✅ 完整 |
| 第二款 | 近30/60/90日起迄漲跌幅 > ±100/130/160% | ✅ 完整 |
| 第三款 | 近6日漲跌異常 + 當日量放大≥5倍 | ✅ 完整 |
| 第四款 | 近6日漲跌異常 + 當日週轉率≥10% | ✅ 完整（需流通股數）|
| 第九款 | 近6日均量及當日量較60日均量放大≥5倍 | ✅ 完整 |
| 第十款 | 近6日累積週轉率>50% 且當日≥10% | ✅ 完整（需流通股數）|
| 第十一款 | 近6日起迄兩日收盤價差≥100元（高價股級距加碼） | ✅ 完整 |
| 第五/六/七/八/十二/十三款 | 券商集中、本益比淨值比、券資比、TDR溢折價、借券賣出、當沖比 | ⚠️ 框架（需額外明細資料，標示「資料不足」）|

處置認定（要點第六條）：連續3個營業日／最近5日內5次／最近10日內6次／
最近30日內12次 達注意標準即處置；第一次與第二次（含以上）措施不同（撮合頻率、預收款券）。

## 重要前提與近似（已在輸出標示）

* **市場平均**：各款「與全體有價證券平均值之差幅」以代理值近似（預設 0，即近似為個股
  自身漲跌幅）。6 日市場平均通常接近 0，故**絕對門檻**為主要拘束條件。
* **類股平均**：「與同類有價證券平均值之差幅」需全類股資料，本工具不檢核（視為成立），
  結果偏「較容易觸發」（保守示警）。
* **第三階反推**建立在「市場／類股平均維持不變、無除權息等非交易因素」假設上。
* **上櫃（TPEx）**：日成交與公告可抓；個股流通股數未串接，週轉率相關款（四、十）以估算為主。
* **興櫃**：採**不同**的注意／處置標準（無漲跌幅限制、標準另訂），本工具會偵測並標示，
  不硬套上市櫃公式。

## 程式結構（標準呼叫鏈）

```
start.py                 入口：uvicorn 啟動 webapp:app
twse_watch/
  webapp.py              FastAPI 後端（/ 與 /api/query）
  core.py                三階段引擎：串接以下模組
  source.py              資料層：TWSE/TPEx 抓取、市場別偵測、民國日期/中文數字解析
  indicators.py          指標：累積漲跌幅、週轉率、量能放大、起迄價差…
  rules.py               13 款注意規則與門檻（Thresholds）
  dispo.py               處置認定（第一、二階）：官方公告 + 距處置次數
  reverse.py             第三階反推：明日臨界股價/週轉率/量能
static/index.html        前端查詢頁（單檔，無需建置）
tests/test_core.py       單元測試
```

> 註：套件內若另有 `engine.py`／`disposition.py`／`server.py`／`run.py`，為開發過程於
> 特定檔案系統覆寫受限而保留的舊版，**已不被引用**，可逕行刪除；標準入口一律為
> `start.py`（→ `webapp` → `core` → `dispo`）。
